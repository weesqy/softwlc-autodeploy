# SoftWLC Autodeploy

Набор Bash-сценариев для автоматизированного развёртывания контроллера беспроводной сети **Eltex SoftWLC** и сопутствующих компонентов на учебном стенде. Сценарии разворачивают серверную часть (SoftWLC + приложение проверки лабораторных работ) и клиентскую часть (EMS-апплет) на ОС семейства Debian/Ubuntu и Astra Linux.

Цель методики — сократить ручной труд оператора: вместо примерно 20 последовательных команд развёртывание сводится к запуску 2 сценариев (по числу команд, а не времени выполнения).

## Архитектура стенда

Стенд разворачивается в VirtualBox и состоит из сервера и одного или нескольких клиентов.

**Сервер** (Ubuntu Server):
- SoftWLC + EMS, MySQL, RADIUS, DHCP — порт **8080** (раздача JNLP-файла EMS-апплета);
- `labcheck` — Node.js-приложение проверки лабораторных работ — порт **9090**.

**Клиент** (Debian/Ubuntu или Astra Linux):
- OpenJDK **17** + IcedTea-Web **1.8.8** — для запуска EMS-апплета по протоколу JNLP.

Приложение `labcheck` использует базы данных SoftWLC (`wireless`, `radius`) на `localhost`, поэтому устанавливается на тот же сервер и только после запуска SoftWLC.

## Состав репозитория

| Сценарий | Узел | Назначение |
|---|---|---|
| `install_softwlc_server.sh` | сервер | Установка контроллера SoftWLC (EMS, MySQL, RADIUS, DHCP) через установщик Eltex |
| `install_labcheck_server.sh` | сервер | Развёртывание Node.js-приложения проверки ЛР, служба systemd, порт 9090 |
| `install_ems_ubuntu-debian.sh` | клиент | EMS-апплет на Debian/Ubuntu: OpenJDK 17 + IcedTea-Web, запуск апплета |
| `install_ems_astra-up.sh` | клиент | EMS-апплет на Astra Linux Common Edition 2.12 «Орёл» |

Файл приложения `app.js` для `labcheck` **не входит в публичный репозиторий** и передаётся отдельно (см. ниже).

## Требования

- **Сервер:** Ubuntu Server 22.04 LTS / 24.04 LTS.
- **Клиенты:** семейство Debian/Ubuntu (онлайн-режим — Ubuntu 20.04+, Debian 11+; более старые версии — через офлайн-режим) и Astra Linux Common Edition 2.12 «Орёл» (x86-64).
- Права суперпользователя (`sudo`).
- Для онлайн-режима — доступ в Интернет; для офлайн-режима — заранее скачанные архивы (см. раздел «Режимы установки»).

## Порядок развёртывания

После клонирования сделайте сценарии исполняемыми либо запускайте их через `sudo bash <сценарий>`:

```bash
git clone https://github.com/weesqy/softwlc-autodeploy.git
cd softwlc-autodeploy
chmod +x *.sh
```

### 1. Сервер: контроллер SoftWLC

```bash
sudo ./install_softwlc_server.sh
# при необходимости URL установщика можно переопределить аргументом:
sudo ./install_softwlc_server.sh https://<адрес>/install_softwlc.sh
```

По завершении EMS-апплет доступен по адресу `http://<IP-сервера>:8080/ems/jws`. Учётные данные администратора выводит установщик Eltex.

### 2. Сервер: приложение проверки лабораторных работ

Запускается после установки SoftWLC — сценарий ожидает готовности EMS на порту 8080.

```bash
# онлайн (app.js передаётся аргументом либо выбирается интерактивно):
sudo bash install_labcheck_server.sh /путь/к/app.js

# офлайн (из распакованного архива рядом со сценарием):
sudo bash install_labcheck_server.sh
```

Создаётся служба systemd `labcheck`, приложение доступно по адресу `http://<IP-сервера>:9090`.

Переменные окружения (необязательные):

| Переменная | Назначение |
|---|---|
| `MODE=online\|offline\|auto` | Принудительный выбор режима (по умолчанию `auto`) |
| `APP_URL=http://...` | Внутренний источник `app.js` (онлайн) |
| `REINSTALL=1` | Переразвернуть начисто (удалить файлы в `/opt/labcheck`) |
| `SKIP_EMS_WAIT=1` | Пропустить ожидание SoftWLC (только для проверки механики) |

### 3. Клиент: EMS-апплет

**Debian/Ubuntu:**

```bash
# интерактивно (IP сервера вводится с клавиатуры, режим выбирается в диалоге):
sudo ./install_ems_ubuntu-debian.sh

# либо адрес сервера передаётся аргументом:
sudo ./install_ems_ubuntu-debian.sh http://192.168.1.23:8080/ems/jws
```

**Astra Linux Common Edition 2.12:**

```bash
sudo ./install_ems_astra-up.sh
sudo ./install_ems_astra-up.sh http://192.168.1.23:8080/ems/jws
```

Сценарий устанавливает OpenJDK 17 и IcedTea-Web 1.8.8, настраивает их альтернативами, загружает `ems_gui.jnlp` с сервера и запускает апплет. На Astra Linux JDK 17 всегда ставится из архива, так как в штатном репозитории нет подходящей версии.

## Режимы установки

Клиентские сценарии и `labcheck` поддерживают два режима, выбираемых интерактивно или переменной окружения:

- **Онлайн** — компоненты загружаются из сети (NodeSource для Node.js 20, репозиторий ОС для OpenJDK, релиз IcedTea-Web с GitHub).
- **Офлайн** — для изолированной сети, компоненты берутся из заранее скачанных локальных архивов:
  - клиент — архив JDK 17 (`*.tar.gz`) и `icedtea-web-1.8.8.linux.bin.zip`;
  - `labcheck` — переносимый Node.js и `node_modules` в составе архива (каталоги `runtime/` и `node_modules/` рядом со сценарием).

## Возможные проблемы

- **Неверное системное время на ВМ** (`Release file ... is not valid yet`). Синхронизируйте время: `sudo timedatectl set-ntp true`.
- **«Пользователь отсутствует в файле sudoers»** (частый случай на Debian, где первый пользователь не входит в группу `sudo`). От имени root: `usermod -aG sudo <пользователь>`, затем перезайти в сеанс.
- **Сервер ещё не вернул JNLP-файл.** После перезагрузки сервера запуск SoftWLC занимает несколько минут; клиентский сценарий повторяет запрос автоматически.
- **Офлайн-архив `labcheck` несовместим с сервером.** Нативный модуль `bcrypt` привязан к паре «архитектура процессора + версия Node», поэтому архив нужно собирать на Linux x86_64 той же версией Node 20, что входит в комплект.
